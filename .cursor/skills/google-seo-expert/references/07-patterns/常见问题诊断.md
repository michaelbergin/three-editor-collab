# Google SEO 常见问题诊断模式

基于Google官方SEO文档的常见问题诊断模式库。

---

## 📉 搜索流量下降诊断

### 模式：流量突然下降

**症状识别：**
- 搜索流量在特定时间点突然大幅下降
- 排名位置明显下滑
- 收录页面数量减少

**诊断步骤：**

1. **确认下降时间点**
   - 检查Google Search Console的性能报告
   - 对比流量下降与Google算法更新时间
   - 查看网站变更日志

2. **分析下降范围**
   - 全站下降 → 可能是算法更新或惩罚
   - 特定栏目下降 → 内容质量问题
   - 特定关键词下降 → 竞争加剧或相关性降低

3. **检查技术问题**
   - robots.txt是否意外阻止爬虫
   - 是否误加了noindex标签
   - 服务器是否返回大量5xx错误
   - 网站是否被黑客攻击

**解决方案：**
- 参考: `05-monitoring-debugging/debugging-search-traffic-drops.md`
- 使用Search Console检查手动操作通知
- 检查Core Web Vitals评分变化

## 🔍 网站收录问题

### 模式：页面不被收录

**症状识别：**
- URL检查工具显示"URL不在Google索引中"
- 新发布页面长期不被收录
- 收录数量远低于实际页面数

**诊断步骤：**

1. **检查可抓取性**
   ```
   问题清单：
   □ robots.txt是否阻止？
   □ 页面是否有noindex标签？
   □ 是否返回404/403/500状态码？
   □ 是否需要登录才能访问？
   ```

2. **检查可发现性**
   ```
   问题清单：
   □ 是否在站点地图中提交？
   □ 是否有内链指向？
   □ 链接是否是JavaScript生成的？
   □ 页面是否在robots.txt允许的范围内？
   ```

3. **检查内容质量**
   ```
   问题清单：
   □ 内容是否原创？
   □ 是否有实质性价值？
   □ 是否被标记为重复内容？
   □ 页面加载速度是否过慢？
   ```

**解决方案：**
- 参考: `02-crawling-indexing/block-indexing.md` (反向检查)
- 使用`ask-google-to-recrawl.md`请求重新抓取
- 检查`sitemaps-overview.md`确保正确提交

## 🔄 重复内容问题

### 模式：多个URL指向相同内容

**症状识别：**
- Search Console显示"重复页面没有规范标签"
- 同一内容有多个URL版本（带参数、不同路径等）
- 排名分散在多个相似页面上

**常见场景：**
```
场景1: URL参数
  example.com/product?id=123
  example.com/product?id=123&color=red

场景2: 协议/域名
  http://example.com/page
  https://example.com/page
  www.example.com/page

场景3: 移动端
  example.com/page
  m.example.com/page
```

**解决方案：**

1. **设置Canonical标签**
   ```html
   <link rel="canonical" href="https://example.com/page" />
   ```
   参考: `02-crawling-indexing/consolidate-duplicate-urls.md`

2. **使用301重定向**
   - 将所有变体重定向到规范URL
   参考: `02-crawling-indexing/301-redirects.md`

3. **配置URL参数**
   - 在Search Console中处理URL参数

## 📱 移动端优化问题

### 模式：移动排名低于桌面

**症状识别：**
- 移动端排名明显低于桌面端
- Search Console显示移动可用性问题
- Core Web Vitals移动评分较低

**诊断清单：**
```
移动友好性检查：
□ 是否使用响应式设计？
□ 视口meta标签是否正确？
□ 字体大小是否可读（至少12px）？
□ 点击元素间距是否足够？
□ 是否有侵入式插页广告？
□ 页面内容是否被截断？

移动性能检查：
□ LCP是否 < 2.5s？
□ FID是否 < 100ms？
□ CLS是否 < 0.1？
□ 是否阻塞渲染的JavaScript？
□ 图片是否优化？
```

**解决方案：**
- 参考: `02-crawling-indexing/mobile-sites-mobile-first-indexing.md`
- 使用PageSpeed Insights检测问题
- 实施AMP加速（可选）: `02-crawling-indexing/amp.md`

## 🏷️ 结构化数据问题

### 模式：富媒体摘要不显示

**症状识别：**
- 实施了结构化数据但富媒体摘要不出现
- Search Console显示结构化数据错误
- Rich Results测试工具显示问题

**诊断步骤：**

1. **验证JSON-LD语法**
   ```bash
   # 使用Google Rich Results测试
   https://search.google.com/test/rich-results
   ```

2. **检查必需属性**
   - 每种类型的Schema都有必需属性
   - 检查是否缺少必填字段

3. **常见错误：**
   ```
   ❌ 拼写错误: "@type": "Prodct"
   ❌ 缺少逗号
   ❌ 属性嵌套错误
   ❌ 使用了不支持的类型组合
   ```

**类型特定诊断：**

| 类型 | 常见问题 | 检查文档 |
|------|---------|---------|
| Product | 缺少price/review | `04-structured-data/product.md` |
| Article | 缺少author/date | `04-structured-data/article.md` |
| Recipe | 缺少image/prepTime | `04-structured-data/recipe.md` |
| FAQPage | 问题少于2个 | `04-structured-data/faqpage.md` |
| Event | 缺少startDate/location | `04-structured-data/event.md` |

## ⚡ 页面速度问题

### 模式：Core Web Vitals不达标

**症状识别：**
- Search Console显示Core Web Vitals警告
- PageSpeed Insights评分低
- 用户体验报告加载缓慢

**诊断矩阵：**

| 指标 | 目标值 | 常见问题 | 解决方案 |
|------|--------|---------|---------|
| LCP | <2.5s | 大图、慢服务器 | 优化图片、使用CDN |
| FID/INP | <200ms | JS执行阻塞 | 代码分割、延迟加载 |
| CLS | <0.1 | 无尺寸图片/字体 | 预留空间、预加载 |
| TTFB | <800ms | 服务器响应慢 | 优化后端、边缘缓存 |
| FCP | <1.8s | 渲染阻塞资源 | 内联关键CSS |

**优化优先级：**
```
P0 (立即)：
- 压缩图片（WebP格式）
- 启用浏览器缓存
- 压缩文本资源（Gzip/Brotli）

P1 (本周)：
- 删除未使用的JavaScript
- 延迟加载非关键资源
- 优化服务器响应时间

P2 (本月)：
- 实施代码分割
- 使用Service Worker缓存
- 优化第三方脚本加载
```

## 🔗 链接问题

### 模式：内部链接结构问题

**症状识别：**
- 深层页面难以被发现
- PageRank分布不均
- 爬虫预算浪费在低价值页面

**诊断检查：**
```
内部链接审计：
□ 重要页面是否在3次点击内？
□ 是否有孤立页面（无内链）？
□ 是否有死链或重定向链？
□ 锚文本是否描述性？
□ 面包屑导航是否完善？

外部链接审计：
□ 是否有质量低劣的入站链接？
□ 出站链接是否标记nofollow（如需要）？
□ 是否参与链接购买/交换？
```

**解决方案：**
- 参考: `02-crawling-indexing/links-crawlable.md`
- 建立清晰的网站架构
- 使用面包屑导航: `04-structured-data/breadcrumb.md`

## 🚨 手动操作惩罚

### 模式：收到Google手动操作通知

**症状识别：**
- Search Console显示手动操作通知
- 搜索排名突然消失或大幅下降
- 网站被标记为违规

**常见手动操作类型：**

| 类型 | 原因 | 恢复时间 |
|------|------|---------|
| 非自然链接 | 购买链接、链接交换 | 数周至数月 |
| 单薄内容 | 低质量、自动生成内容 | 数周 |
| 隐藏内容 | cloak、隐藏文本 | 数周 |
| 用户生成内容垃圾 | 评论垃圾、论坛垃圾 | 数天至数周 |
| 结构化数据垃圾 | 标记与内容不符 | 数天 |

**恢复步骤：**
```
1. 仔细阅读违规通知
2. 全面清理违规内容/链接
3. 记录所有修复措施
4. 提交重新审核请求
5. 耐心等待处理结果
```

**参考：**
- `05-monitoring-debugging/spam-policies.md`
- Google Search Console帮助

## 🎯 优先级决策矩阵

当同时面临多个SEO问题时，按以下优先级处理：

```
紧急且重要（立即）：
├─ 网站完全无法访问
├─ 收到手动操作惩罚
├─ 重要页面被noindex
└─ 大面积404错误

重要但不紧急（本周）：
├─ Core Web Vitals优化
├─ 移动端适配问题
├─ 结构化数据实施
└─ 重复内容处理

紧急但不重要（排期）：
├─ 单个页面排名下降
├─ 低流量页面优化
└─ 非核心功能调整

不紧急不重要（观察）：
├─ 微调元描述
├─ 图片alt优化（已优化主要图片）
└─ 微调内部链接锚文本
```
