## 2026 年 3 月

### 为论坛和问答页面标记添加了新的支持属性

内容：为论坛和问答页面标记添加了更多受支持的属性。

原因：为了让 Google 提取系统更清楚地了解评论串结构。
          这能避免我们在处理论坛和问答内容时产生误解。

### 在漫游器元标记文档中添加了备注

内容：在漫游器元标记文档中添加了一条备注，说明 Google 搜索如何处理 HTML 标头之外的漫游器元标记。

原因：行为未发生变化，但之前未记录在文档中。

### 移除了有关 JavaScript 和无障碍功能的过时信息

内容：从 JavaScript SEO 基础知识文档中移除了有关无障碍功能的章节。

原因：这些信息已过时，不如以前那么有用。Google 搜索多年来一直能够渲染 JavaScript，所以用 JavaScript 加载内容并不会“让 Google 搜索更难”抓取。现在，大多数辅助技术也能够与 JavaScript 搭配使用。

### 添加了首选图片最佳实践

内容：在图片 SEO 最佳实践和探索文档中添加了有关使用元数据指定首选图片的新部分。

原因：根据用户反馈，我们明确指出，在确定 Google 搜索和 Google 探索中的图片缩略图时，Google 会同时使用 schema.org 标记和 og:image meta 标记作为来源。

```
og:image
```

```

```

内容：我们在 JavaScript 文档和整合重复网址最佳实践中添加了关于 JavaScript 规范化最佳实践的部分。

原因：规范化发生在渲染之前和之后，因此务必尽可能清晰地指定规范网址。对于 JavaScript，这意味着将规范网址设置为与原始 HTML 中的网址相同的网址；如果无法做到这一点，则不要在原始 HTML 中包含规范网址。

### 澄清了 noindex 和 JavaScript

内容：在我们的 JavaScript 文档中，我们明确说明了 Google 的抓取工具如何处理使用 JavaScript 的网页中的 noindex 标记。

```
noindex
```

原因：虽然 Google 也许能够渲染使用 JavaScript 的网页，但这种行为的定义并不明确，并且可能会发生变化。如果您确实希望将网页编入索引，请勿在原始网页代码中使用 noindex 标记。

```
noindex
```
